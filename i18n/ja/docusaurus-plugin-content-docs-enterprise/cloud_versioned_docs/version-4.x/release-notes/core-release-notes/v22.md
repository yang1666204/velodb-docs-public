# v2.2

この記事では、VeloDB Core v2.2のリリースノートについて説明します。

> **Note** VeloDB Core v2.2は[Apache Doris v1.2](https://doris.apache.org/docs/releasenotes/v1.2/release-1.2.0)をベースに開発されました。

## v2.2.34 (June 1, 2023)  

バグ修正

- オブジェクトストレージサイズの不整合によりインポートがスタックする問題を修正しました。

## v2.2.33 (May 15, 2023)  

バグ修正

- **JDBC URL**または**mysql -D**をクラスターと組み合わせて使用した際にアクセス不能になる問題を修正しました。

## v2.2.32 (April 27, 2023)

改善

- **file cache**メトリクスを追加しました。

バグ修正

- [Apache Doris v1.2.4](https://doris.apache.org/docs/releasenotes/v1.2/release-1.2.4)の複数のバグ修正を同期しました。
- **segment file size**の順序が正しくない問題を修正しました。

## v2.2.31 (April 19, 2023)

新機能

- 外部リソース（HMS、external stage）設定時の**dryrun**テストをサポートしました。
- **Varchar**カラムの長さを変更する**lightweight schema change**をサポートしました。
- 分割された圧縮ファイルの**copy into**インポート方式をサポートしました。

改善

- **routine load**を無効化しました。
- データセキュリティを強化しました。

バグ修正

- **top n**クエリに関する問題を修正しました。
- **file cache**の除去によるクエリ失敗の問題を修正しました。
- 大量の**tablets**を一度にインポートすると失敗する可能性がある問題を修正しました。
- **inverted index**のメモリ不安定性の問題を修正しました。
- **inverted index**がクエリ失敗を引き起こす可能性がある問題を修正しました。
- **copy into**実行中にファイルが上書きされ、インポートがスタックする問題を修正しました。
- 同名のクラスターをリネームすることによる「no backend」の問題を修正しました。

## v2.2.21 (March 23, 2023)

バグ修正

- **cumulative compaction**がトリガーされない可能性がある問題を修正しました。
- 純粋な計算プランノードがクラスター間をまたぐ可能性がある問題を修正しました。
- **rename cluster**実行後の名前不整合を修正しました。
- **drop cluster**実行時に**schema change job**が自動的に失敗できない問題を修正しました。
- **delete predicate**により**base compaction**が失敗する問題を修正しました。
- **create table like**の問題を修正しました。

## v2.2.20 (February 28, 2023)

改善

- 複数の**cache**最適化を実装しました。
- **copy into**インポートのスループットを改善しました。
- 空テーブルのクエリパフォーマンスを最適化しました。

バグ修正

- **add inverted index**実行時のデータが正しくないバグを修正しました。
- **parquet**インポートがスタックする可能性がある問題を修正しました。
- [Apache Doris v1.2.4](https://doris.apache.org/docs/releasenotes/v1.2/release-1.2.4)のデータレイク分析に関する複数の問題を修正しました。

## v2.2.1 (February 9, 2023)

新機能

- **external stage**のクロスアカウント認証をサポートしました。

改善

- **copy into**のアップロード速度を最適化しました。
- クラスターおよびステージ権限表示のための**show roles**を最適化しました。

バグ修正

- 計算ノードがスタックする可能性がある問題を修正しました。
- **temp partition**によって引き起こされる可能性があるリソースリークの問題を修正しました。
- **admin**ユーザー以外が**show cluster**を実行できない問題を修正しました。

## v2.2.0 (January 17, 2023)

新機能

- **String**型の**inverted index**をサポートしました。
- **numeric**および**date**型の**BKD multidimensional numerical index**をサポートしました。
- **array**型の**inverted index**および**BKD multi-dimensional numerical index**をサポートしました。
- スキーマを動的に変更する**dynamic table**をサポートしました。

改善

- 増分**ADD/DROP INDEX**を最適化しました。
- **Index searcher cache**を最適化しました。
- **Top n**クエリ（**ORDER BY c1, c2 LIMIT n**）のパフォーマンスを最適化しました。

バグ修正

- インデックスフィルタリング時に元のカラムを読み取らない問題を修正しました。
