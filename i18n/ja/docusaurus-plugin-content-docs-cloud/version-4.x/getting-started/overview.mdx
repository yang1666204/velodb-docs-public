# はじめに

VeloDB Cloudは、Apache Dorisをベースとした新世代のマルチクラウドネイティブリアルタイムデータウェアハウスであり、エンタープライズレベルのビッグデータのリアルタイム分析ニーズを満たすことに焦点を当て、お客様に極めて費用対効果が高く、使いやすいデータ分析サービスを提供します。

VeloDB Cloudはお客様に公開されています。お客様がVeloDB データウェアハウスをAWS（Amazon Web Services）、Microsoft Azure、GCP（Google Cloud Platform）にデプロイしたい場合は、[VeloDB Cloud](https://www.velodb.cloud/passport/login)にアクセスしてログインしてください。



## 主要機能

- **極限パフォーマンス**: ストレージにおいて、VeloDB Cloudは効率的な列型ストレージとデータインデックスを採用しており、コンピューティングにおいて、VeloDB CloudはMPP分散コンピューティングアーキテクチャとX64およびARM64向けに最適化されたベクトル化実行エンジンに依拠しています。VeloDB CloudはClickBench公開パフォーマンス評価において世界最高レベルにあります。
- **費用対効果**: VeloDB Cloudはストレージとコンピューティングを分離したクラウドネイティブアーキテクチャを採用し、クラウドインフラストラクチャに基づいて設計・開発されています。ストレージにおいて、共有オブジェクトストレージが極めて低いコストを実現し、コンピューティングにおいて、VeloDB Cloudはオンデマンドスケーリングと開始・停止をサポートし、リソース利用率を最大化します。
- **使いやすさ**: ワンクリック デプロイメント、すぐに使用可能。MySQL互換ネットワーク接続プロトコルをサポート。Kafka/Flink/Spark/DBTとの統合コネクタを提供。強力で使いやすいビジュアル運用保守管理コンソールとデータ開発ツールを備えています。
- **単一統合**: 単一の製品で複数の分析ワークロードを実行できます。リアルタイム/インタラクティブ/バッチコンピューティングタイプ、構造化/半構造化データタイプ、および外部データレイク（Hive、Iceberg、Hudiなど）とデータベース（MySQL、Elasticsearchなど）の連携分析をサポートします。
- **オープン**: オープンソースのApache Dorisの研究開発に基づき、VeloDB Cloudはオープンソースコミュニティへのイノベーションの貢献を続けています。VeloDB CloudはApache Dorisシンタックスプロトコルと完全に互換性があり、Apache Dorisとデータを自由に移行できます。国内外のエコシステム製品やツールとの互換性と相互認証を継続します。国内外のクラウドプラットフォームとのオープンな協力により、製品はマルチクラウドで動作し、一貫したユーザー体験を提供します。
- **安全かつ安定**: データセキュリティにおいて、VeloDB Cloudは完全な権限制御、データ暗号化、バックアップと回復メカニズムを提供します。運用保守管理において、VeloDB Cloudは包括的なオブザーバビリティメトリクス収集とデータウェアハウスサービスのビジュアル管理を提供します。技術サポートにおいて、VeloDB Cloudは完全なチケット管理システムとリモートアシスタンスプラットフォームを備え、複数レベルのエキスパートサポートサービスを提供します。



## 主要概念

![key concepts of velodb](/images/cloud/key-concepts-of-velodb.jpeg)

<div style={{textAlign: 'center'}}>VeloDB Cloudの主要概念</div>

<br />

- **Organization**: Organizationは企業または比較的独立したグループを表し、ユーザーはVeloDB Cloudに登録後、組織としてサービスを利用できます。OrganizationはVeloDB Cloudにおける請求および決済オブジェクトであり、異なる組織間では請求、リソース、およびデータが相互に分離されています。
- **Warehouse**: Warehouseはコンピューティングとストレージリソースを含む論理的概念です。各組織は注文、広告、物流などの異なる事業のデータ分析ニーズを満たすために複数のwarehouseを作成できます。同様に、異なるwarehouse間でもリソースとデータは相互に分離されており、組織内のセキュリティ要件を満たすために使用できます。
- **Cluster**: Clusterはwarehouse内のコンピューティングリソースであり、1つまたは複数のコンピューティングノードを含み、弾力的にスケーリング可能です。Warehouseは複数のclusterを含むことができ、これらは基盤となるデータを共有します。異なるclusterは統計レポート、インタラクティブ分析などの異なるワークロードを満たすことができ、複数cluster間のワークロードは相互に干渉しません。
- **Storage**: 成熟した安定したオブジェクトストレージシステムを使用して全データを保存し、マルチコンピューティングcluster共有ストレージをサポートします。これによりデータウェアハウスに極めて低いストレージコスト、高いデータ信頼性、ほぼ無制限のストレージ容量をもたらし、上位コンピューティングclusterの実装複雑性を大幅に簡素化します。



## 製品アーキテクチャ

![velodb cloud architecture](/images/cloud/velodb-cloud-architecture.jpeg)

<div style={{textAlign: 'center'}}>クラウドネイティブストレージ・コンピューティング分離アーキテクチャ</div>
<br />

- **Cloud Service Layer**: クラウドサービス層は、VeloDB Cloudが提供するサポートサービスの集合であり、認証、アクセス制御、クラウドインフラストラクチャ管理、メタデータ管理、クエリ解析と最適化などを含み、「warehouse」の形で表現されます。Warehouseは相互に分離されています。
- **Computing Cluster Layer**: コンピューティング層はストレージ層から分離されており、柔軟な弾力スケーリングとスムーズなアップグレードをサポートします。コンピューティング層はいくつかのコンピューティングclusterで構成されます。複数のコンピューティングclusterがストレージを共有し、複数cluster間でワークロードが分離されます。各clusterは1つまたは複数のコンピューティングノードを含みます。コンピューティングノードは高速ハードディスクを使用してホットデータキャッシュ（Cache）を構築し、先進的なクエリオプティマイザーと豊富なインデックス技術により不要なコールドデータの読み取りを回避し、オブジェクトストレージの高い応答遅延の問題を大幅に最適化し、お客様に究極のデータ分析パフォーマンスを提供します。
- **Shared Storage Layer**: VeloDB Cloudの最下層は安価で高可用性、ほぼ無限にスケーラブルなオブジェクトストレージを共有ストレージ層として使用し、オブジェクトストレージに基づいて深い最適化設計を行っており、お客様のデータ分析コストを数倍削減し、PBレベルのデータ分析ニーズを容易にサポートできます。異なるクラウド環境におけるオブジェクトストレージの統一された標準と成熟度は、マルチクラウドにおけるVeloDB Cloudの一貫した使用体験も強化しています。



## アプリケーションシナリオ

- **高同時実行リアルタイムレポートと分析**: VeloDB Cloudを使用してオンライン高同時実行レポートを処理し、リアルタイム、高速、安定、高可用性サービスを取得します。リアルタイムデータ書き込み、サブ秒クエリ応答、高同時実行ポイントクエリをサポートし、clusterの高可用性デプロイメント要件を満たします。
- **ユーザープロファイルと行動分析**: VeloDB Cloudに基づいてユーザーCDP（Customer Data management Platform）データウェアハウスプラットフォーム階層を構築し、ミリ秒レベルの列追加と動的Tableをサポートしてビジネスの変化に柔軟に対応し、豊富な行動分析機能をサポートして開発を簡素化し効率を向上させ、高レベルの直交ビットマップをサポートしてプロファイルシーンで秒レベルの人物特定を実現します。
- **ログストレージと分析**: VeloDB Cloudデータウェアハウスをログシステムに統合してリアルタイムログクエリ、低コストストレージ、効率的な処理を実現し、企業ログシステムの全体コストを削減し、ログシステムのパフォーマンスと信頼性を向上させます。
- **レイクハウス統合と連携分析**: データレイク、データベース、データウェアハウスを単一プラットフォームに統一統合し、VeloDB Cloudのデータ連携クエリ加速機能に依拠して、高パフォーマンスビジネスインテリジェンスレポート、Adhoc分析、増分ETL/ELTデータ処理サービスを提供します。



## Apache Dorisとの関係

VeloDB Inc（「**VeloDB**」）は、Apache Dorisに基づく製品を持つ商用会社です。VeloDBは、Apache Dorisの創設チームによって2023年5月に設立されました。VeloDBはApache Dorisの重要な推進力です。7名のPMCメンバーと20名のCommitterを擁し、Apache Dorisの一連のコアバージョンのリリースを主導してきました。VeloDBはオープンソースApache Dorisを積極的に推進し、技術はオープンソースユーザーと開発者に利益をもたらし、Apache Dorisに基づく商用製品を立ち上げ、ビジネスは商用顧客を支援し、二輪駆動でオープンソースとビジネスの健全な成長を実現しています。

VeloDB CloudはVeloDBがApache Dorisに基づいて構築した新世代のマルチクラウドネイティブリアルタイムデータウェアハウスです。Apache Dorisと比較して、VeloDB Cloudには以下の主な違いがあります：

- コアバージョンがより成熟し安定しており、より多くのエンタープライズレベル機能とクラウドネイティブ機能を備えています。
- 内蔵のビジュアル運用保守管理コンソールとデータ開発ツールを提供し、ユーザーがインストール・デプロイする必要がなく、すぐに使用でき、最小限の運用保守と管理を実現しています。
