# v4.0

この記事では、VeloDB Core v4.0のリリースノートについて説明します。

> **注意** VeloDB Core v4.0は[Apache Doris v3.0](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.0)をベースに開発されました。

## v4.0.9 (2025年12月10日)

バグ修正

- UDFで間欠的に発生するNoClassDefFoundErrorの問題を修正しました。
- パーティション列のないIceberg外部Tableをクエリする際の不正なクエリ結果を修正しました。

## v4.0.8 (2025年10月23日)

バグ修正

- Elasticsearchカタログをクエリする際にジョブ挿入タスクが失敗する問題を修正しました。
- 不正なJSONパスによって引き起こされるクラッシュを修正しました。

## v4.0.7 (2025年9月15日)

改善

- [Apache Doris v3.0.8](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.8)のすべての機能、改善、およびバグ修正をマージしました。

## v4.0.6 (2025年8月17日)

改善

- [Apache Doris v3.0.6](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.6)のすべての機能、改善、およびバグ修正をマージしました。
- バージョン3.0.xから4.0.xへのアップグレードプロセスを高速化しました。
- マルチクラスターシナリオでのクエリパフォーマンスと可用性を最適化し、システムの可用性を向上させました。
- TopN文のクエリパフォーマンスを向上させました。

バグ修正

- tabletキャッシュの早期クリーンアップによって引き起こされるクエリパフォーマンスの問題を修正しました。

## v4.0.5 (2025年5月13日)

改善

- [Apache Doris v3.0.4](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.4)と[Apache Doris v3.0.5](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.5)のすべての機能、改善、およびバグ修正をマージしました。
- 早期データ回収の問題に対処することで可用性を向上させました。
- group commitオペレーションのCPU使用率を最適化しました。

バグ修正

- v4.0.4からv4.0.5にアップグレードする際の転置インデックス再構築中のコアダンプを解決しました。
- recyclerがtxn labelsを回収する速度が遅い問題を修正しました。
- スキーマ変更中のバケット列順序の変更によって引き起こされるデータ正確性の問題を修正しました。
- ネストされたキャスト: cast(cast(datetimev2 as datev2) as datetimev2)によって引き起こされる不正な結果を修正しました。
- 複数文字の行区切り文字が検出された際の同時CSVインポート中の潜在的なデータ損失を修正しました。
- 集合交差関数でのスタックオーバーフローの問題を修正しました。

## v4.0.4 (2024年12月6日)

改善

- [Apache Doris v3.0.3](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.3)のすべての機能、改善、およびバグ修正をマージしました。

バグ修正

- tabletキャッシュの早期クリーンアップによって引き起こされるクエリパフォーマンスの問題を修正しました。
- ファイルキャッシュでの過剰な小さなファイルの問題を修正しました。
- copy into observerオペレーションでのメモリリークを修正しました。

## v4.0.3 (2024年10月11日)

改善

- [Apache Doris v3.0.2](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.2)のすべての機能、改善、およびバグ修正をマージしました。
- 非常に大きなトランザクションのインポートをサポートしました（デフォルトでは無効）。

バグ修正

- カタログの再生がbackendのwarehouse起動を失敗させる可能性がある問題を修正しました。

## v4.0.2 (2024年8月25日)

改善

- [Apache Doris v3.0.1](https://doris.apache.org/docs/releasenotes/v3.0/release-3.0.1)のすべての機能、改善、およびバグ修正をマージしました。

バグ修正

- カタログの再生がwarehouseの再開を失敗させる可能性がある問題を修正しました。

## v4.0.1 (2024年7月9日)

新機能

- [v3.0.x](./v30.md)（x >= 9）からv4.0.x-betaへのVeloDB Coreアップグレードをサポートしました。

改善

- [Apache Doris v3.0.0 rc11](https://github.com/apache/doris/tree/3.0.0-rc11)のすべての機能、改善、およびバグ修正をマージしました（[Apache Doris v2.1](https://doris.apache.org/docs/releasenotes/v2.1/release-2.1.0)のすべての機能を含む）。
- [Apache Doris v3.0.0 rc09](https://github.com/apache/doris/tree/3.0.0-rc09)のすべての機能、改善、およびバグ修正をマージしました。

バグ修正

- VeloDB Core v3.0.9からアップグレード後に、クラスター権限を復元するためにbackgroundでwarehouseの再起動が必要になる問題を修正しました。
- 逆引きインデックスクエリがスタックする可能性がある問題を修正しました。
- s3プロトコルを使用してAzure Blob Storageにインポートする際に過剰にファイルがインポートされる問題を修正しました。

## v4.0.0 (2024年5月15日)

新機能

- マルチTableマテリアライズドビュー（**MTMV**）、透明な書き換え高速化、自動更新、外部Tableから内部Tableへのマテリアライズドビュー、およびマテリアライズドビューの直接クエリをサポートしました。この機能に基づいて、マテリアライズドビューはデータウェアハウスの階層モデリング、ジョブスケジューリング、データ処理にも使用できます。
- 新しい**Variant**および**IP**データ型をサポートし、一連の分析関数を改善することで、複雑な半構造化データの保存と分析を容易にしました。
- 自動増分列、自動パーティショニング、およびサーバー上でバッチリクエストを蓄積してから集約送信する機能をサポートし、大規模データのリアルタイム書き込み効率を向上させました。
- **Arrow Flight**に基づく高速読み取りインターフェースをサポートし、データ転送効率を**100倍**向上させることで、データサイエンスやその他の形式の大規模データ読み取りシナリオに容易に対応できるようになりました。

改善

- [Apache Doris v2.1.0](https://doris.apache.org/docs/releasenotes/v2.1/release-2.1.0)のすべての機能、改善、およびバグ修正をマージしました。
- ブラインドアウトオブボックスクエリのパフォーマンスを大幅に向上させ、チューニングなしで良好なパフォーマンスを実現しました。複雑なSQLクエリパフォーマンスのさらなる改善を含み、**TPC-DS** 1TBテストデータセットで**100%**以上のパフォーマンス向上を達成し、クエリパフォーマンスは業界最前線にあります。
- データレイク分析のパフォーマンスを大幅に向上させ、**Trino**および**Spark**と比較してそれぞれ**4から6倍**のパフォーマンス向上を実現しました。また、マルチSQL方言互換性を導入することで、ユーザーが元のシステムからVeloDB Cloudデータウェアハウスにシームレスに切り替えることを容易にしました。
