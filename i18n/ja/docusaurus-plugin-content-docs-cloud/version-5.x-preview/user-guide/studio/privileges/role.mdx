# Role

接続でロールを管理するか、ロールを認可することができます。
Admin権限を持つユーザーのみが他のロールを追加および変更できます。
VeloDB Studioは現在、ロールを通じてロール配下のユーザーの管理をサポートしていません。これは、ユーザーを作成または変更する際にロールを指定する必要があることを意味します。


:::warning
adminとoperatorロールは変更できません。表示のみ可能です。
:::

Table内のユーザー項目は、そのロールに付与されたユーザーを示しており、Apache Dorisが現在ロールの作成時刻をサポートしていないため、作成時刻は空になっています。ただし、将来のバージョンでのサポートに備えて表示スペースを予約しています。
