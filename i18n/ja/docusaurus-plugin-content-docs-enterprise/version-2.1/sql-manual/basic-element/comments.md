## 説明

コメントはアプリケーションを読みやすく保守しやすくすることができます。たとえば、ステートメント内にコメントを含めて、アプリケーション内でそのステートメントの目的を説明することができます。SQLステートメント内のコメント（HINTを除く）は、ステートメントの実行に影響しません。HINTを使用したこの特定の形式のコメントの使用に関する情報については、HINTセクションを参照してください。

SQLステートメントでは、コメントは任意のキーワード、パラメータ、または句読点の間に表示できます。ステートメント内にコメントを含める方法は2つあります：

- 複数行コメント：スラッシュとアスタリスク（`/*`）でコメントを開始し、その後にコメントテキストを続けます。このテキストは複数行にわたることができます。アスタリスクとスラッシュ（`*/`）でコメントを終了します。開始文字と終了文字とテキストの間にスペースや改行を入れる必要はありません。
- 単一行コメント：2つのダッシュ（`--`）でコメントを開始し、その後にコメントテキストを続けます。このテキストは新しい行に拡張することはできません。コメントは改行文字で終了します。

## 例

### 複数行コメント

```sql
SELECT /* This is a multi-line comment
          that can span multiple lines */
       column_name
FROM   table_name;
```
### 単一行コメント

```sql
SELECT column_name -- This is a single-line comment
FROM   table_name;
```
