## 説明

`CANCEL REBALANCE DISK`文は、Backend（BE）ノードの高優先度ディスクデータバランシングをキャンセルするために使用されます。この文には以下の機能があります：

- 指定されたBEノードの高優先度ディスクバランシングをキャンセルできます。
- クラスター全体のすべてのBEノードの高優先度ディスクバランシングをキャンセルできます。
- キャンセル後、システムはデフォルトのスケジューリング方法を使用してBEノードのディスクデータのバランシングを継続します。

## 構文

```sql
ADMIN CANCEL REBALANCE DISK [ ON ( "<host>:<port>" [, ... ] ) ];
```
## Optional Parameters

**1. `"<host>:<port>"`**

> 高優先度ディスクバランシングをキャンセルする必要があるBEノードのリストを指定します。
>
> 各ノードはホスト名（またはIPアドレス）とハートビートポートで構成されます。
>
> このパラメータが指定されていない場合、すべてのBEノードの高優先度ディスクバランシングをキャンセルします。

## Access Control Requirements

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持っている必要があります：

| Privilege       | Object      | Notes                                         |
| :-------------- | :---------- | :-------------------------------------------- |
| ADMIN           | System      | このコマンドを実行するためには、ユーザーはADMIN権限を持っている必要があります。 |

## Usage Notes

- このステートメントは、システムが指定されたBEのディスクデータのバランシングを優先しなくなることのみを示します。ただし、システムはデフォルトのスケジューリング手法を使用してBEのディスクデータをバランシングし続けます。
- このコマンドを実行した後、以前に設定された高優先度バランシング戦略は即座に無効になります。

## Examples

- クラスター内のすべてのBEの高優先度ディスクバランシングをキャンセル：

    ```sql
    ADMIN CANCEL REBALANCE DISK;
    ```
- 指定されたBEに対する高優先度ディスクバランシングをキャンセルする：

```sql
ADMIN CANCEL REBALANCE DISK ON ("192.168.1.1:1234", "192.168.1.2:1234");
```
