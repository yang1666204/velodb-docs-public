# セキュリティと信頼性

:::info
セキュリティと信頼性はDesktopバージョンでのみ有効です。Serverバージョンはプライベートネットワークまたは VPC にデプロイされ、VeloDB サーバーとの通信は行われません。
:::

VeloDB Studio では、セキュリティを最も重要な優先事項として位置づけています。基本的な統計情報であるユーザー数/バージョン機能を除き、VeloDB サーバーにデータを送信することはありません。

VeloDB Studio のアクセス中にデータが安全であることを確保するため、SSH トンネルを通じてデータベースサーバーへの接続を確立することを推奨します。

## データ収集

製品の改善を支援するため、匿名の使用データを VeloDB サーバーに送信します。機密データは一切収集しません。
データ収集にはオープンソースの `Matomo` サービスを使用しており、`Matomo` の詳細情報は[こちら](https://matomo.org/)で確認できます。Google Analytics と同様です。

主に以下のデータを収集します：

- アプリの起動
- オペレーティングシステムのバージョン番号
- オペレーティングシステムの種類
- オペレーティングシステムの言語

これらの項目は、アプリを使用しているユーザー数や、ユーザーが使用しているオペレーティングシステムの種類などの基本情報を理解するのに役立ちます。

すべての接続とデータ、その他の情報は収集および送信されません。`Help -> Open Console -> Network` から送信するすべてのデータを確認でき、これは透明性を保っています。

上記のデータ収集について懸念がある場合は、フィードバックをお寄せください。将来のリリースでデータ収集を無効にするオプションの提供を検討いたします。

## セキュリティアップデート

VeloDB Studio は既知のセキュリティ脆弱性を修正するため、定期的にアップデートされます。
