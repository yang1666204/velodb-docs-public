## Description

このステートメントは、BEノードの基本ステータス情報を表示するために使用されます。

## Syntax

```sql
 SHOW BACKENDS
```
## 戻り値

| Column                      | Note                                                                                                                                                                                                                                                           |
|-------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BackendId               | 現在のBEのID。                                                                                                                                                                                                                                      |
| Host                    | 現在のBEのIPアドレスまたはホスト名。                                                                                                                                                                                                                 |
| HeartbeatPort           | 現在のBEのハートビートサービスの通信ポート。                                                                                                                                                                             |
| BePort                  | 現在のBEのthrift RPC通信ポート。                                                                                                                                                                                                           |
| HttpPort                | 現在のBEのHTTP通信ポート。                                                                                                                                                                                                                 |
| BrpcPort                | 現在のBEのbRPC通信ポート。                                                                                                                                                                                                                 |
| ArrowFlightSqlPort      | 現在のBEのArrowFlightプロトコルの通信ポート。                                                                                                          |
| LastStartTime           | 現在のBEが開始されたタイムスタンプ。                                                                                                                                                                                                                     |
| LastHeartbeat           | 現在のBEが送信した最後の成功したハートビートのタイムスタンプ。                                                                                                         |
| Alive                   | 現在のBEが生きているかどうか。                                                                                                                                                                                                               |
| SystemDecommissioned    | この値がtrueの場合、現在のBEノードが安全な廃止プロセス中であることを意味する。                                                                          |
| TabletNum               | 現在のBEに保存されているタブレット数。                                                                                                                                                                                                                |
| DataUsedCapacity        | 現在のBEのデータが占有しているディスク容量。                                                                                                                                                                                         |
| TrashUsedCapacity       | 現在のBEのゴミ箱内のデータが占有しているディスク容量。                                                                                                            |
| AvailCapacity           | 現在のBEの利用可能なディスク容量。                                                                                                                    |
| TotalCapacity           | 現在のBEの総ディスク容量。TotalCapacity = AvailCapacity + TrashUsedCapacity + DataUsedCapacity + その他の非ユーザーデータファイルが占有するディスク容量。                                                                                               |
| UsedPct                 | 現在のBEの総使用済みディスク容量の割合。                                                                                                                 |
| MaxDiskUsedPct          | 現在のBEの全ディスクの中で使用済みディスク容量の最大割合。                                                                               |
| RemoteUsedCapacity      | 現在のBEがホットコールド階層化機能を使用した後にリモートストレージにアップロードされたデータが占有するディスク容量。                                                                                                          |
| Tag                     | 現在のBEのタグ情報で、JSON形式で表示される。現在のBEリソースグループの名前が保存されている。                                                                           |
| ErrMsg                  | 現在のBEのハートビートが失敗した時のエラーメッセージ。                                                                                                  |
| Version                 | 現在のBEのバージョン情報。                                                                                                                     |
| Status                  | 現在のBEの一部のステータス情報で、JSON形式で表示される。これには以下が含まれる：lastSuccessReportTabletsTime、lastStreamLoadTime、isQueryDisabled、isLoadDisabledなど。異なるバージョンで保存される情報は若干異なる場合があることに注意してください。 |
| HeartbeatFailureCounter | 現在のBEの連続したハートビート失敗回数。この数がFE Masterによって設定された`max_backend_heartbeat_failure_tolerance_count`（デフォルト値は1）を超えた場合、`Alive`フィールドはfalseに設定される。                         |
| NodeRole                | 現在のBEの役割。2種類ある：`mix`はデフォルトの役割で、`computation`は現在のノードが連合分析クエリにのみ使用されることを意味する。                                         |

## アクセス制御要件

このSQLを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| Privilege  | Object | Notes |
|------------|----|----|
| ADMIN_PRIV |    |    |

## 使用上の注意事項

クエリ結果をさらにフィルタリングする必要がある場合、テーブル値関数[backends()](../../../sql-functions/table-valued-functions/backends.md)を使用できます。SHOW BACKENDSは以下のステートメントと同等です：

```sql
SELECT * FROM BACKENDS();
```
