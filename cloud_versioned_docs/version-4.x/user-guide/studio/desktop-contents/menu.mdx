# Application Menu


## Connect to the database

Click the `+` button to enter the database connection information, click the `Save` button to save the connection, and click the card to connect to the database.

## Restart

Click the `VeloDB Studio` menu and click `Restart` to restart `VeloDB Studio`.

## Reload

Overloading refers to reloading the `VeloDB Studio` application, similar to a browser refreshing a page. You can use the reload function when you encounter a page stutter.

Click the `VeloDB Studio` menu, click `Reload App` or `⌘(Command) + R` (Windows apps use `CTRL + R`) to overload `VeloDB Studio`.


## Update

Click the `VeloDB Studio` menu and click `Check for updates` to check for updates. The new version will be downloaded automatically, and it will be prompted to install after the download is completed. (There will be a prompt after the download is ready)

## Quit

Click the `VeloDB Studio` menu and click `Exit` to exit `VeloDB Studio`. After exiting, `VeloDB Studio` will close all connections and all its own processes, and will not run in the background.

## Uninstall

<Tabs>
    <TabItem value='Mac' label='Mac'>
        1. Drag VeloDB Studio from the `Applications` folder to the Trash to uninstall.
        2. The data path of `VeloDB Studio` is located in `/Users/[your username]/Library/Application Support/velodb-studio`. The path will not be deleted during uninstallation. If you need to delete the path, you can delete it manually. But after deletion, the data of `VeloDB Studio` will be lost.
        3. The log path of `VeloDB Studio` is located in `/Users/[your username]/Library/Application Support/velodb-studio/logs`. The path will not be deleted during uninstallation. If you need to delete the path, you can delete it manually.
    </TabItem>
    <TabItem value='Windows' label='Windows'>
        1. Click the uninstall button in the Windows system settings to uninstall VeloDB Studio.
        2. The data path of `VeloDB Studio` is located in `C:\Users\[your username]\AppData\Roaming\velodb-studio`. The path will not be deleted during uninstallation. If you need to delete the path, you can delete it manually. But after deletion, the data of `VeloDB Studio` will be lost.
        3. The log path of `VeloDB Studio` is located in `C:\Users\[your username]\AppData\Roaming\velodb-studio\logs`. The path will not be deleted during uninstallation. If you need to delete the path, you can delete it manually.
    </TabItem>
</Tabs>
