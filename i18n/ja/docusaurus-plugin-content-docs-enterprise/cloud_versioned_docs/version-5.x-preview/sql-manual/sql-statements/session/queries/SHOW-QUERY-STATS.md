## Description

このステートメントは、履歴クエリによってヒットしたデータベーステーブルカラムを表示するために使用されます

## Syntax

```sql
SHOW QUERY STATS [ { [FOR <db_name>] | [FROM <table_name>] } ] [ALL] [VERBOSE]];
```
## オプションパラメータ

**1. `<db_name>`**

> このパラメータが設定されている場合、データベースのマッチング情報が表示されます

**2. `<table_name>`**

> このパラメータが設定されている場合、テーブルのマッチングステータスがクエリされます

**3. `ALL`**

> ALLは、すべてのインデックスのマッチング情報を表示するかどうかを指定します

**4. `VERBOSE`**

> VERBOSEは詳細なマッチング情報を表示します

## アクセス制御要件

このSQLコマンドを実行するユーザーは、少なくとも以下の権限を持つ必要があります：

| 権限              | オブジェクト | 注記                                  |
|:------------------|:-------------|:-------------------------------------|
| SELECT_PRIV       | DATABASE     | クエリ対象のデータベースにSELECT権限を持つ必要があります |

## 使用上の注意

- データベースとテーブルの過去のクエリマッチングステータスをクエリします。データはfeが再起動された後にリセットされ、統計は各feごとに収集されます。

- FOR DATABASEとFROM TABLEを使用して、クエリするデータベースまたはテーブルのマッチング情報を指定でき、その後にデータベース名またはテーブル名を続けます。

- ALLとVERBOSEパラメータは単独で使用することも組み合わせて使用することも可能ですが、最後に使用する必要があり、テーブルクエリのみに適用されます。

- データベースが使用されていない場合、単に'SHOW QUERY STATS'を実行するだけで、すべてのデータベースのヒット数が表示されます。

- ヒット結果には2つのカラムが存在する可能性があります：QueryCountはカラムがクエリされた回数を示し、FilterCountはカラムがwhere条件としてクエリされた回数を示します。
