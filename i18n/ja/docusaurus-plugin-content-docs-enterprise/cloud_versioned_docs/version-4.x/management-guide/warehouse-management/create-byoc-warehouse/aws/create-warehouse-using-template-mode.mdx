# Template Modeを使用したBYOCウェアハウスの作成

このガイドでは、**Template Mode**を使用してVeloDBウェアハウスを作成するプロセスを説明します。

1. VeloDB Cloud Consoleにログインします。
2. コンソールにログイン後、左上角の「Create New Warehouse」をクリックし、BYOCを選択して、Template Modeを選択してセットアップガイドに入ります。

## Step 1: ウェアハウスの設定

1. 基本設定のセットアップ
   1. ウェアハウス名を入力します。サポートされる文字には、中国語文字、英字（大文字小文字を区別しない）、数字、アンダースコア（_）、ハイフン（-）が含まれます。
   2. Cloud Platform: AWSを選択
   3. ウェアハウスがデプロイされるAWSリージョンを選択します。
   4. **VPC**については、既存のものまたはプラットフォームに一度も登録されたことがない新しいvpcを選択します。
2. 初期クラスターリソース設定
   - Compute (vCPU)
     - デフォルト: 8 vCPU
     - 調整可能範囲: クラスターあたり8–1024 vCPU
     - より高い容量の要求: ワークロードが1024 vCPUを超える必要がある場合は、クォータ承認についてサポートにお問い合わせください。
   - Cache
     - 動的サイジング: キャッシュ容量の制限は、選択されたコンピュートサイズに基づいて自動的にスケールされます。
     - 上限/下限: 最小および最大キャッシュ閾値は、vCPU割り当てに比例して変化します。
3. 初期ユーザー設定
    デフォルトのadminユーザーのパスワードを設定できます。パスワードは以下の要件を満たす必要があります：
     - 長さ: 8–20文字
     - 許可される文字: 大文字と小文字、数字、および特殊文字（例：`~!@#$%^&*()_+|<>,.?/:;'[]{}"` ）
     - 複雑さ: 少なくとも3つの文字タイプを含む必要があります（例：文字 + 数字 + 記号）。

    > **NOTE**
    > adminユーザーは自動的にプロビジョニングされ、完全なシステム権限を持つウェアハウス管理者として機能します。クラスターが作成された後、ウェアハウス設定ページに移動してadminユーザーのパスワードをリセットできます。

すべての必須フィールドを入力した後、Nextボタンをクリックして続行します。

## Step 2: AWSリソースの準備

このプロセスでは、必要なAWSリソースを作成するAWS CloudFormationテンプレートの起動をガイドします。すべてのリソースは単一のスタックで管理されます：

- EC2セキュリティグループ。これは、関連するAWSリソースへの、およびそこからのトラフィックを制御する仮想ファイアウォールです。
- S3バケット。これはウェアハウスデータを格納します。
- IAMロール。これらには、ウェアハウスが関連するAWSリソースにアクセスすることを許可する権限が付与されます。

> **Note**
> CloudFormationスタックのデプロイメントを開始するには、管理者権限が必須です。これらの権限がない場合は、管理者に連絡して続行してください。権限が不十分な場合、デプロイメントが失敗します。

VeloDBが提供するリソースオーケストレーションテンプレートは、あなたのクラウドアカウント下で実行され、テンプレートコードはオープンです。
以下のリンクを通じて、VeloDBが提供するリソースオーケストレーションテンプレートを取得できます：

> [https://velodb-cloud-online.s3.us-east-1.amazonaws.com/public/aws-byoc.yaml](https://velodb-cloud-online.s3.us-east-1.amazonaws.com/public/aws-byoc.yaml)

CloudFromationスタックの完了後、VeloDBコンソールに戻って進行中のウェアハウスデプロイメントを確認します。このプロセスは通常約5分かかります。完了するまでお待ちください。
