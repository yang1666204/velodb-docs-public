# v2.4

この記事では、VeloDB Core v2.4のリリースノートについて説明します。

> **注意** VeloDB Core v2.4は[Apache Doris v1.2](https://doris.apache.org/docs/releasenotes/v1.2/release-1.2.0)をベースに開発されました。

## v2.4.6 (2023年11月14日)

バグ修正

- 複数のコンピュートノードのメモリリークを修正しました。

## v2.4.5 (2023年11月6日)

改善

- バランス戦略を最適化しました。

バグ修正

- **replace tmp partition**によって引き起こされるクエリ失敗の問題を修正しました。
- **pthread**と**bthread**の混在によって引き起こされるデッドロックの問題を修正しました。

## v2.4.4 (2023年9月20日)

改善

- VeloDB Core v2.3.9およびv2.3.10のすべての機能、改善、バグ修正をマージしました。
- インポート機能の使いやすさを向上させました。

バグ修正

- 大量のパーティションを含む**truncate table**をリクエストする際のmysqlセッションの応答なしの問題を修正しました。

## v2.4.3 (2023年9月11日)

新機能

- 料金滞納によるサービス停止をサポートしました。

改善

- VeloDB Core v2.3.8のすべての機能、改善、バグ修正をマージしました。
- http文字列を含むs3 load endpointとの互換性を追加しました。

バグ修正

- docker環境のタイムゾーンの問題を修正しました。

## v2.4.2 (2023年8月28日)

新機能

- **copy into**インポート方法で**定数値を設定**することをサポートしました。

改善

- ストレージ情報統計の精度を向上させました。

バグ修正

- クラウド**stream load**が失敗する可能性がある問題を修正しました。

## v2.4.1 (2023年8月3日)

新機能

- リクエストを受信した際のオンデマンドクラスタ**自動開始**をサポートしました。
- **stream load**をサポートしました。

改善

- VeloDB Core v2.3.5、v2.3.6、v2.3.7のすべての機能、改善、バグ修正をマージしました。

バグ修正

- **inverted index**クエリが失敗する可能性がある問題を修正しました。

## v2.4.0 (2023年7月3日)

新機能

- **session variable**を通じてSQLがキャッシュを使用するかどうかを制御することをサポートしました。
- クラスタ間でのインポートデータの増分データ同期をサポートしました。
- テーブルクラスタに基づく**キャッシュデータのプリヒート**機能をサポートしました。
- SQLを通じて**キャッシュ使用量**を照会することをサポートしました。
- アイドル時のオンデマンドクラスタ**自動停止**をサポートしました。

改善

- データセキュリティを強化しました。
- クエリがキャッシュをミスした際の応答速度を向上させました。

バグ修正

- **set default cluster**が失敗する可能性がある問題を修正しました。
- アップグレードプロセスが失敗する可能性がある問題を修正しました。
