# v2.0

この記事では、VeloDB Core v2.0のリリースノートについて説明します。

> **Note** VeloDB Core v2.0は[Apache Doris v1.1](https://doris.apache.org/docs/releasenotes/v1.1/release-1.1.0/)をベースに開発されました。

## v2.0.15 (January 5, 2023)

新機能

- **rename warehouse**をサポートしました。

改善

- **copy into delete sign**を最適化しました。
- stageを作成するために**copy into**を実行する際に、オブジェクト情報が有効かどうかを自動的に判定する機能をサポートしました。
- **partition level tablet balance**を最適化しました。
- **cache**設定パラメータタイプのチェックを追加しました。
- **tablet sink profile**を最適化しました。
- 権限システムを改善しました。

バグ修正

- **sync_tablet_rowsets**における**stale request**の誤判定問題を修正しました。
- **load job**リーク問題を修正しました。
- **partition level balance**のバグを修正しました。

## v2.0.14 (December 14, 2022)

新機能

- **select**ステートメントで**hint**を通じて**set cluster**をサポートしました。
- 各Tableに対して複数の**copy job**を保持することをサポートしました。

改善

- **s3**へのインポートおよび書き込みのパフォーマンスを最適化しました。
- インポートされた**Segment index**がオブジェクトストレージにアップロードされた後、事前に**read cache**に入力されるようになりました。

バグ修正

- **parquet**をインポートする際に計画と実行のデータ型が一致しない問題を修正し、ログインポートの失敗を解決しました。
- クラスター間でのインポート問題を修正しました。

## v2.0.13 (December 2, 2022)

新機能

- **copy into**ステートメントの追加条件**delete on**をサポートしました。
- **show data**ステートメントでウェアハウス全体のデータ量を表示することをサポートしました。

改善

- **show processlist**ステートメントの使用を復元しました。
- 削除されたstageのリサイクル速度を向上しました。
- **cache**データのディスクへの書き込み速度を向上しました。

バグ修正

- **drop cluster**実行時のウェアハウス動作の不整合問題を修正しました。

## v2.0.12 (November 26, 2022)

改善

- **create stage**実行時にオブジェクトストレージ情報の有効性を検証することをサポートしました。

バグ修正

- **write cache**が早期に削除される可能性がある問題を修正しました。
- **cache**がヒットしない可能性がある問題を修正しました。
- 非colocateTableの**tablets**が不均等に分散される可能性がある問題を修正しました。
