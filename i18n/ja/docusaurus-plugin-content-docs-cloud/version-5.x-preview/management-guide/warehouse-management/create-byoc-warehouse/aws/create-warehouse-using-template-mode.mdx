# Template Modeを使用したBYOCウェアハウスの作成

このガイドでは、**Template Mode**を使用してVeloDBウェアハウスを作成するプロセスについて説明します。

1. VeloDB Cloud Consoleにログインします。
2. コンソールにログイン後、左上の「Create New Warehouse」をクリックし、BYOCを選択してTemplate Modeを選択し、セットアップガイドに進みます。

## ステップ 1: ウェアハウスの設定

1. 基本設定を行います
   1. ウェアハウス名を入力します。サポートされる文字：漢字、文字（大文字小文字区別なし）、数字、アンダースコア（_）、ハイフン（-）
   2. Cloud Platform：AWSを選択
   3. ウェアハウスをデプロイするAWSリージョンを選択します。
   4. **VPC**については、既存のものまたはプラットフォームに未登録の新しいvpcを選択します。
2. 初期クラスターリソース設定
   - Compute（vCPU）
     - デフォルト：8 vCPU
     - 調整可能範囲：クラスターあたり8–1024 vCPU
     - より高い容量のリクエスト：ワークロードが1024 vCPUを超える必要がある場合は、クォータ承認についてサポートにお問い合わせください。
   - Cache
     - 動的サイジング：キャッシュ容量制限は、選択されたコンピュートサイズに基づいて自動的にスケールされます。
     - 上限/下限：最小および最大キャッシュしきい値は、vCPU割り当てに比例して変化します。
3. 初期ユーザー設定
    デフォルトのadminユーザーのパスワードを設定できます。パスワードは以下の要件を満たす必要があります：
     - 長さ：8–20文字
     - 使用可能文字：大文字と小文字、数字、特殊文字（例：`~!@#$%^&*()_+|<>,.?/:;'[]{}""`）
     - 複雑さ：少なくとも3つの文字種類を含む必要があります（例：文字+数字+記号）。

    > **NOTE**
    > adminユーザーは自動的にプロビジョニングされ、システム全体の権限を持つウェアハウス管理者として機能します。クラスターが作成された後、ウェアハウス設定ページに移動してadminユーザーのパスワードをリセットできます。

すべての必須フィールドを入力した後、Nextボタンをクリックして続行します。

## ステップ 2: AWSリソースの準備

このプロセスでは、必要なAWSリソースを作成するAWS CloudFormationテンプレートの起動について説明します。すべてのリソースは単一のスタックで管理されます：

- EC2セキュリティグループ：関連するAWSリソースとの間のトラフィックを制御する仮想ファイアウォールです。
- S3バケット：ウェアハウスデータを格納します。
- IAMロール：ウェアハウスが関連するAWSリソースにアクセスできる権限が付与されます。
- VPC Private Endpoint：VeloDB Cloud Serviceとのプライベート通信を確立します。
- EC2インスタンス：VeloDB Agentをホストし、サービスオーケストレーションを担当します。

> **Note**
> CloudFormationスタックのデプロイを開始するには、管理者権限が必須です。これらの権限がない場合は、管理者に連絡して進行してください。権限が不十分な場合、デプロイは失敗します。

VeloDBが提供するリソースオーケストレーションテンプレートはクラウドアカウント下で実行され、テンプレートコードはオープンです。
VeloDBが提供するリソースオーケストレーションテンプレートは、以下のリンクから取得できます：

> [https://velodb-cloud-online.s3.us-east-1.amazonaws.com/public/aws-byoc.yaml](https://velodb-cloud-online.s3.us-east-1.amazonaws.com/public/aws-byoc.yaml)

CloudFromationスタックの完了後、VeloDBコンソールに戻り、進行中のウェアハウスデプロイを確認します。このプロセスは通常約5分かかります。完了までお待ちください。
