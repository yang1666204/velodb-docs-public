# AWS Glue Icebergカタログへの接続

このガイドでは、ビジュアルインターフェースを使用してVeloDB CloudをAWS Glue Icebergカタログに接続する手順を説明します。

:::warning
**前提条件**: 続行する前に、[AWS Setup Guide](./aws-glue/setup)を完了してS3バケット、Glueデータベース、適切な権限を持つIAMユーザーを作成してください。
:::

## Step 1: Catalogsに移動

VeloDBウェアハウスで、左サイドバーの**Development**の下にある**Data**セクションを探し、**Catalogs**をクリックします。

**Add External Catalog**ボタンがあるCatalogsページが表示されます。

![Catalogs Page](/images/cloud/integration/aws-glue/03-catalogs-page.png)

## Step 2: External Catalogの追加

**Add External Catalog**をクリックして、利用可能なカタログタイプを表示します。

![Add External Catalog Options](/images/cloud/integration/aws-glue/04-add-external-catalog-options.png)

利用可能なカタログタイプには以下があります:
- **AWS Glue** - Glue管理のIceberg/Hiveテーブル用
- **Hive Metastore** - オンプレミスHive用
- **Amazon S3 Tables** - S3ネイティブテーブル用
- **Iceberg REST Catalog** - Polaris、Gravitino、Tabular用
- **Apache Polaris** - Polarisカタログ用

## Step 3: AWS Glueの選択

**AWS Glue**をクリックして設定フォームを開きます。

![AWS Glue Form](/images/cloud/integration/aws-glue/05-aws-glue-catalog-form.png)

## Step 4: カタログ設定の構成

[AWS Setup Guide](./aws-glue/setup)の値を使用してフォームを入力します:

### 基本情報

| Field | Description | Example |
|-------|-------------|---------|
| **Catalog Name** | カタログの一意識別子 | `glue_iceberg_useast1` |
| **Comment** | オプションの説明 | `AWS Glue Iceberg catalog` |

### Metastore設定

| Field | Description | Example |
|-------|-------------|---------|
| **Table Format** | `Iceberg`または`Hive`を選択 | `Iceberg` |
| **Warehouse** | IcebergウェアハウスへのS3パス | `s3://my-bucket/iceberg` |
| **AWS Glue Region** | GlueカタログのAWSリージョン | `us-east-1` |
| **AWS Glue Endpoint** | Glue APIエンドポイント | `https://glue.us-east-1.amazonaws.com` |

### 認証

| Field | Description |
|-------|-------------|
| **Authentication Type** | `Access Key`または`IAM Role` |
| **AK (Access Key)** | AWSアクセスキーID |
| **SK (Secret Key)** | AWSシークレットアクセスキー |

### ストレージ

S3アクセスに同じ認証情報を使用するため、**"Use the authentication details configured for Metastore access"**にチェックを入れたままにします。

:::info
AKとSKフィールドには、[AWS Setup Guide](./aws-glue/setup)の実際のAWS認証情報を入力してください。
:::

## Step 5: カタログの作成

**Confirm**をクリックしてカタログを作成します。

VeloDBがAWS Glueへの接続を検証します。成功すると、成功通知が表示され、カタログがリストに表示されます。

![Catalog Created Successfully](/images/cloud/integration/aws-glue/10-glue-catalog-created-success.png)

## Step 6: SQLでの確認

左サイドバーの**Query > SQL Editor**に移動して、カタログ接続をテストします。設定を確認するために以下のクエリを実行してください（`your_catalog_name`と`your_database`を実際の名前に置き換えてください）:

### 全カタログの一覧表示

```sql
SHOW CATALOGS;
```
期待される出力: リスト内に新しいGlueカタログが表示されるはずです。

### カタログ内のデータベースを一覧表示する

```sql
-- Replace 'your_catalog_name' with your catalog name (e.g., glue_iceberg_useast1)
SHOW DATABASES FROM your_catalog_name;
```
### データベース内のテーブル一覧を取得する

```sql
-- Replace with your catalog and database names
SHOW TABLES FROM your_catalog_name.your_database;
```
### Iceberg テーブルからのデータクエリ

```sql
-- Example: Query the sample table created in Setup Step 5
SELECT * FROM your_catalog_name.your_database.your_table LIMIT 10;
```
:::info
外部でデータが更新された後にテーブルが表示されない場合は、`REFRESH CATALOG your_catalog_name;` を実行してメタデータキャッシュをクリアしてください。
:::

## トラブルシューティング

| 問題 | 解決方法 |
|-------|----------|
| "Cannot convert namespace to Glue database name" | データベース名にハイフンが含まれています。アンダースコアのみを使用してください。 |
| "Lake Formation permission denied" | AWS CLI経由でLake Formationの権限を付与してください。AWS Setup Guideを参照してください。 |
| "Connection timeout" | VeloDBウェアハウスとAWS Glueが同じリージョンにあることを確認してください。 |
| 外部でデータが更新された後の"Table does not exist" | `REFRESH CATALOG catalog_name;` を実行してメタデータキャッシュをクリアしてください。 |
